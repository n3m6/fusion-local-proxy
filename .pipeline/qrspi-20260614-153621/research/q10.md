#### `tsconfig.json` — required compiler options for strict mode + ESM + Node 20

A minimal working `tsconfig.json` requires these settings:

| Compiler Option | Required Value | Rationale |
|---|---|---|
| `strict` | `true` | Enables all strict type-checking flags (`strictNullChecks`, `strictFunctionTypes`, `strictBindCallApply`, `strictPropertyInitialization`, `noImplicitAny`, `noImplicitThis`, `alwaysStrict`, `useUnknownInCatchVariables`) |
| `target` | `"ES2023"` or `"ES2022"` | Node.js 20 supports ES2023 (V8 11.3); `ES2022` is the safe floor. Must not be lower than `ES2020` for ESM dynamic `import()` syntax |
| `module` | `"NodeNext"` | Required for Node.js native ESM (`"type": "module"` in `package.json`). Alternative: `"ESNext"` works with `moduleResolution: "NodeNext"` |
| `moduleResolution` | `"NodeNext"` | Required to resolve ESM imports with `.js` extensions and `package.json` `"exports"` field per Node.js ESM spec |
| `esModuleInterop` | `true` | Allows default imports from CJS modules; needed for `openai`, `@anthropic-ai/sdk`, and Hono interop |
| `skipLibCheck` | `true` | Skips type-checking of `.d.ts` files (SDKs, node types) for faster compilation; does not affect project source strictness |
| `forceConsistentCasingInFileNames` | `true` | Prevents casing mismatches on case-insensitive filesystems |
| `resolveJsonModule` | `true` | Required to import `fusion.config.json` from TypeScript source |
| `isolatedModules` | `true` | Required by `tsx` (which uses `esbuild` under the hood, transpiling files in isolation) |
| `outDir` | `"./dist"` | Conventional output directory for `tsc` builds |
| `rootDir` | `"./src"` | Source root, matching the planned `src/` layout |
| `declaration` | `true` | Optional but conventional for library-style projects; emits `.d.ts` files |
| `sourceMap` | `true` | Enables runtime debugging by mapping compiled JS back to TS source |
| `types` | `["node"]` | Includes Node.js 20 type definitions (`@types/node`) |

**Additional strictness flags** (already covered by `"strict": true`, listed for explicitness): `noUnusedLocals`, `noUnusedParameters`, `exactOptionalPropertyTypes`, `noUncheckedIndexedAccess` are optional but conventional in strict codebases.

#### `tsconfig.json` — minimal example

```json
{
  "compilerOptions": {
    "target": "ES2023",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "declaration": true,
    "sourceMap": true,
    "types": ["node"]
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

#### `package.json` — required fields and scripts

| Field | Required Value | Rationale |
|---|---|---|
| `type` | `"module"` | Opts the entire project into Node.js ESM mode; enables `import`/`export` syntax; `.js` files treated as ES modules |
| `scripts.dev` | `"tsx src/main.ts"` | Runs the project directly via `tsx` (no pre-compilation). Alternative: `"tsx watch src/main.ts"` for auto-reload during development |
| `scripts.build` | `"tsc"` | Compiles TypeScript to JavaScript in `dist/` |
| `scripts.start` | `"node dist/main.js"` | Runs the compiled output in production |
| `engines.node` | `">=20.0.0"` | Declares the minimum Node.js version; `tsx` and the planned SDKs (`openai`, `@anthropic-ai/sdk`) all support Node 20+ |
| `devDependencies.tsx` | `"^4.x"` | The dev runner. `tsx` uses `esbuild` for instant TypeScript execution without compilation |
| `devDependencies.typescript` | `"^5.x"` | TypeScript compiler. v5.x is required for `module: "NodeNext"` and `moduleResolution: "NodeNext"` |
| `devDependencies.@types/node` | `"^20.x"` | Node.js 20 type definitions for standard library APIs |
| `dependencies` | (various) | `hono`, `@hono/node-server`, `openai`, `@anthropic-ai/sdk`, `zod` — per `requirements.md` and `goals.md` planned dependencies |
