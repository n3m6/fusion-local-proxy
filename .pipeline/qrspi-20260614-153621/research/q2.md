#### SSE Wire Format

The SDK parses standard SSE from the response body. The internal `ServerSentEvent` type (`core/streaming.d.ts:3–7`):
```typescript
type ServerSentEvent = {
    event: string | null;   // the event type (e.g., "message_start")
    data: string;           // JSON payload string
    raw: string[];          // raw SSE lines
};
```

The wire format follows the SSE spec: each event consists of `event:` and `data:` lines separated by double newlines. The `data` field is a JSON string of the event payload object.

#### SSE Event Ordering Constraints

The fixed sequence, derived from the `MessageStream` class (`lib/MessageStream.d.ts`) and `RawMessageStreamEvent` union:

1. **`message_start`** — emitted exactly once, first. Contains the full initial `Message` snapshot.
2. For each content block (zero or more):
   a. **`content_block_start`** — announces the block type and initial fields (`index` ascends from 0).
   b. **`content_block_delta`** — zero or more incremental updates (text, thinking, JSON, citations, signature).
   c. **`content_block_stop`** — signals block completion.
3. **`message_delta`** — emitted once. Contains the final `stop_reason`, `stop_sequence`, and cumulative `usage`.
4. **`message_stop`** — emitted once, final event. Stream ends.

The `MessageStream` convenience class (`lib/MessageStream.d.ts:6–18`) exposes higher-level events (`text`, `thinking`, `inputJson`, `citation`, `signature`, `contentBlock`, `message`, `finalMessage`, `streamEvent`, `error`, `abort`, `end`) that aggregate these raw SSE events.
