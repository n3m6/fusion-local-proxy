### Fix Guidance
1. **Anthropic SSE events**: Update the Purpose column for `src/infrastructure/inbound/http/anthropic/translator.ts` and `.../anthropic/sse-encoder.ts` to reference all 6 event types in the documented sequence: `message_start` → `content_block_start` → `content_block_delta` → `content_block_stop` → `message_delta` → `message_stop`. Match the design's key decision.
2. **SDK version**: Change `@anthropic-ai/sdk` version from `^0.37.0` to `^0.104.1` in both the Slice 1 file-map entry for `package.json` and Convention Note 3.
3. **Test file paths**: Prefix `synthesis-prompt.test.ts`, `judge-prompt.test.ts`, and `analysis-schema.test.ts` with `src/domain/services/` so the file-map entries name full, verifiable paths. Update the Slice 6 diagram node to name each created test file explicitly.
4. **PanelRunner interface**: Change the `PanelRunner` constructor from `chatPort: ChatModelPort` (singular) to `chatPorts: ChatModelPort[]` so the runner receives one port per panel model. Update the `PanelRunner.run()` method to remove `panelModels: ModelRef[]` (the ports already know their models). Alternatively, keep `panelModels: ModelRef[]` for metadata but pair each with its corresponding port from the constructor array. Update the Slice 6 container pseudocode to pass `panelPorts` (the array created by `factory.create()`) into `PanelRunner`.

### Summary
FAIL — four issues: (1) Anthropic SSE event count contradicts the design's explicit 6-event resolution, (2) `@anthropic-ai/sdk` version is wrong, (3) three test file entries lack directory paths, (4) `PanelRunner` cannot fan-out to different backends with a single `ChatModelPort`.
